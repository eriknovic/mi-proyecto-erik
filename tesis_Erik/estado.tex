\chapter{Estado del arte}
\label{cap:soa}
La \textbf{\textit{integración de los recursos de información en una memoria corporativa}} ha sido poco explotada por las organizaciones o áreas de investigación. Existen algunos trabajos sobre la \textit{integración de información} que incorporan a las tecnologías semánticas para modelar su dominio o emplear el concepto de memoria corporativa. Los trabajos que se estudiaron para este \textit{estado del arte}, son aquellos que cumplen con alguno de estos criterios:

\begin{itemize}
\item Integración de los recursos: es el proceso de búsqueda y recuperación de la información sobre los recursos de información. En la sección \ref{sec:intdk} se define formalmente este concepto.
\item Memoria corporativa: es la representación del conocimiento en una organización. En la sección \ref{sec:mecor} se define formalmente este concepto.
\item Modelo semántico: es la representación del conocimiento a partir de las tecnologías semánticas. En la sección \ref{sec:rdf} y \ref{sec:reginf} se describe este concepto.
\item Inferencia en el modelo: es el proceso de deducir información a partir del modelo semántico. En la sección \ref{sec:reginf} se define formalmente este concepto.
\item Interfaz de usuario: es la construcción de un aplicación visual para que los usuarios pregunten o naveguen a partir  del modelo semántico. En la sección \ref{cap:piu} se describen las funciones que debe tener un prototipo para la integración de los recursos.
\end{itemize}

La sección \label{sec:eoaisr} describe los trabajos que se estudiaron para la integración de los recursos. Al final de esta sección, se presenta una tabla comparativa de estos trabajos y los criterios que éstos cumplen.

En este \textit{estado del arte} se contempla un estudio de las aplicaciones para la creación, gestión y modificación de un ontología. Las aplicaciones estudiadas, se agrupan de acuerdo a la funcionalidad de éstas.

\begin{enumerate}
\item Escribir los declaraciones en forma de triple y guardarlos en alguna sintaxis estándar.
\item Escribir los axiomas mediante los vocabularios estándar (OWL y RDF(S)).
\item Gestión del grafo RDF, es decir: carga del modelo, consulta de información e inferencia.
\end{enumerate}

En la sección \ref{sec:eoats} se describen las aplicaciones que se estudiaron para representar, aprovechar y consultar la información en una ontología.

\section{Herramientas para la integración semántica de recursos}
\label{sec:eoats}
Un \textbf{\textit{descriptor de recursos}} \cite{Uren:2006} es una herramienta para transformar la \textit{información explícita de los recursos a tripletas}, en alguna sintaxis (RDF/XML, Turtle, N-triple). El objetivo de un descriptor es construir el ABox para una ontología, para ello, se requiere tener un TBox para saber cuáles clases y propiedades, se pueden emplear al momento de hacer las declaraciones. Un descriptor proporciona una \textit{interfaz gráfica de usuario} (GUI\label{sym:gui}) para simplificar a los usuarios la creación y modificación de las declaraciones. Algunos descriptores proporcionan la generación automatica de declaraciones, a partir de un proceso de aprendizaje en un corpus documental o de imágenes.

\begin{itemize}
\item \textbf{\textit{OntoMat Annotizer}} \cite{OntoBasDoc} es una herramienta para hacer anotaciones semánticas de páginas web, documentos basados en texto plano y lenguajes de marcado\footnote{M. Siroker, ``OntoMat Annotizer,''  Disponible en: \url{http://projects.semwebcentral.org/projects/ontomat/}}. El objetivo de esta herramienta es que el usuario cree de manera amigable instancias y declaraciones de éstas, mediante la funcionalidad de arrastrar y soltar (drag-and-drop).
\item \textbf{\textit{MnM}} \cite{Uren:2006} es una herramienta que proporciona apoyo automatizado y semi-automatizado para describir páginas Web con contenido semántico\footnote{The Open University, ``MnM,''  Disponible en: \url{http://projects.kmi.open.ac.uk/akt/MnM/}}. MnM tiene GUI que integra un editor de ontología, navegador Web, un editor de instancias y de propiedades. El objetivo de esta herramienta es describir documentos a partir de declaraciones derivadas de ontologías preexistentes.
\item \textbf{\textit{GATE}} \cite{Cunningham2011a} es un entorno de desarrollo integrado (IDE\label{sym:ide}) para el desarrollo de componentes en el procesamiento del lenguaje humano y el procesamiento de texto\footnote{The University of Sheffield, ``GATE,''  Disponible en: \url{http://gate.ac.uk/}}. Las tareas en el procesamiento de texto son: \textit{minería web, extracción de información  y descripciones semánticas}.
\item \textbf{\textit{Aktive Media}} \cite{Uren:2006} es una GUI para la descripción automática de una colección de imágenes o documentos (batch annotation) para un contexto específico. ``\textit{El objetivo de Aktive es automatizar el proceso de descripción, mediante la sugerencia del conocimiento al usuario de forma interactiva, mientras éste está describiendo}\footnote{A. Chakravarthy, V. Lanfranchi , F. Ciravegna, ``AKTive Media,''  Disponible en: \url{http://www.aktors.org/technologies/aktivemedia/index.html}}''. Estas sugerencias se hacen con base en axiomas y descripciones previas.
\end{itemize}

La finalidad de estas herramientas es facilitar la generación de descripciones en forma de triple. Sin embargo hay varias razones, por las cuáles, no se elige una de estas herramientas para alcanzar este fin. Estas razones se listan a continuación:

\begin{itemize}
\item Todas estas aplicaciones solo permiten hacer declaraciones de documentos e imágenes, por ello, no proporcionan una solución a la heterogeneidad en formato.
\item Algunas aplicaciones no interpretan los axiomas que están escritos con los vocabularios OWL y RDF(s). Herramientas con este comportamiento son: \textit{OntoMat Annotizer y MnM}.
\item Algunas aplicaciones cambian los prefijos en las tripletas por sus propios prefijos, esto significa un trabajo extra para cambiar estos prefijos con otra aplicación. Herramientas con este comportamiento son: \textit{Aktive Media y GATE}.
\item Algunas aplicaciones no tienen una versión estable. Herramientas con este comportamiento son: \textit{OntoMat Annotizer y MnM}.
\item Algunas aplicaciones no tienen documentación disponible para solucionar problemas de configuración. Herramientas con este comportamiento son: \textit{Aktive Media, GATE y MnM}.
\end{itemize}

La opción que eligió para representar el conocimiento explícito en forma de tripletas, es mediante el uso de scripts. Porque estos scripts facilitan y agilizan el proceso de escritura de tripletas en alguna sintaxis estándar. Un script se escribe en algún lenguaje de programación, con la finalidad de tener un mejor control en la información que se quiere representar.

Aunque estos scripts no poseen una interfaz gráfica para seleccionar la información de los recursos. Estos se puede solucionar mediante el uso de formularios web que capturen la información sobre los recursos. Posteriormente, la información se guarda en algún documento de texto plano, para que los scripts transformen  esta información en triples RDF.\\

Un triplestore es un framework para el almacenamiento y consulta de tripletas RDF. Este triplestore se considera un Sistema Gestor de Base de Datos para datos modelados en RDF. Típicamente, un triplestore emplea el estándar SPARQL como mecanismo para la consulta de datos. Algunos triplestore proporcionan la capacidad de inferir. Esta inferencia se hace a partir de los axiomas de un TBox. Con la  finalidad de que los resultados de las consultas en SPARQL sean más amplios [20].
