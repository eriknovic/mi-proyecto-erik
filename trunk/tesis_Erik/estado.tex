\chapter{Estado del arte}
\label{cap:soa}
La \textbf{\textit{integración de los recursos de información en una memoria corporativa}} ha sido poco explotada por las organizaciones o áreas de investigación. Existen algunos trabajos sobre la \textit{integración de información} que incorporan a las tecnologías semánticas para modelar su dominio o emplear el concepto de memoria corporativa. Los trabajos que se estudiaron para este \textit{estado del arte}, son aquellos que cumplen con alguno de estos criterios:

\begin{itemize}
\item Integración de los recursos: es el proceso de búsqueda y recuperación de la información sobre los recursos de información. En la sección \ref{sec:intdk} se define formalmente este concepto.
\item Memoria corporativa: es la representación del conocimiento en una organización. En la sección \ref{sec:mecor} se define formalmente este concepto.
\item Modelo semántico: es la representación del conocimiento a partir de las tecnologías semánticas. En la sección \ref{sec:rdf} y \ref{sec:reginf} se describe este concepto.
\item Inferencia en el modelo: es el proceso de deducir información a partir del modelo semántico. En la sección \ref{sec:reginf} se define formalmente este concepto.
\item Interfaz de usuario: es la construcción de un aplicación visual para que los usuarios pregunten o naveguen a partir  del modelo semántico. En la sección \ref{cap:piu} se describen las funciones que debe tener un prototipo para la integración de los recursos.
\end{itemize}

La sección \label{sec:eoaisr} describe los trabajos que se estudiaron para la integración de los recursos. Al final de esta sección, se presenta una tabla comparativa de estos trabajos y los criterios que éstos cumplen.

En este \textit{estado del arte} se contempla un estudio de las aplicaciones para realizar la \textit{integración semántica de los recursos en una memoria corporativa}. Las aplicaciones estudiadas, se agrupan de acuerdo a las siguientes funcionalidades.

\begin{enumerate}
\item Escribir los declaraciones en forma de triple y guardarlos en alguna sintaxis estándar.
\item Escribir los axiomas mediante los vocabularios estándar (OWL y RDF(S)).
\item Gestión del grafo RDF, es decir: carga del modelo, consulta de información e inferencia.
\end{enumerate}

En la sección \ref{sec:eoats}, se describen estas aplicaciones con base en su funcionalidad. Al final de cada agrupación, se da a conocer: \textit{cuál herramienta se eligió para facilitar y efectuar la funcionalidad dada}.

\section{Herramientas para la integración semántica de recursos}
\label{sec:eoats}
Un \textbf{\textit{descriptor semánticos de recursos}} \cite{Uren:2006} es una herramienta para transformar la \textit{información explícita de los recursos a tripletas}, en alguna sintaxis (RDF/XML, Turtle, N-triple). El objetivo de un descriptor es construir el ABox para una ontología, para ello, se requiere tener un TBox para saber cuáles clases y propiedades, se pueden emplear al momento de hacer las declaraciones. Un descriptor proporciona una \textit{interfaz gráfica de usuario} (GUI\label{sym:gui}) para simplificar a los usuarios la creación y modificación de las declaraciones. Algunos descriptores proporcionan la generación automatica de declaraciones, a partir de un proceso de aprendizaje en un corpus documental o de imágenes.

En la siguiente lista, se presentan los descriptores semánticos que estudiamos.

\begin{itemize}
\item \textbf{\textit{OntoMat Annotizer}} \cite{OntoBasDoc} es una herramienta para hacer anotaciones semánticas de páginas web, documentos basados en texto plano y lenguajes de marcado\footnote{M. Siroker, ``OntoMat Annotizer,''  Disponible en: \url{http://projects.semwebcentral.org/projects/ontomat/}}. El objetivo de esta herramienta es que el usuario cree de manera amigable instancias y declaraciones de éstas, mediante la funcionalidad de arrastrar y soltar (drag-and-drop).
\item \textbf{\textit{MnM}} \cite{Uren:2006} es una herramienta que proporciona apoyo automatizado y semi-automatizado para describir páginas Web con contenido semántico\footnote{The Open University, ``MnM,''  Disponible en: \url{http://projects.kmi.open.ac.uk/akt/MnM/}}. MnM tiene GUI que integra un editor de ontología, navegador Web, un editor de instancias y de propiedades. El objetivo de esta herramienta es describir documentos a partir de declaraciones derivadas de ontologías preexistentes.
\item \textbf{\textit{GATE}} \cite{Cunningham2011a} es un entorno de desarrollo integrado (IDE\label{sym:ide}) para el desarrollo de componentes en el procesamiento del lenguaje humano y el procesamiento de texto\footnote{The University of Sheffield, ``GATE,''  Disponible en: \url{http://gate.ac.uk/}}. Las tareas en el procesamiento de texto son: \textit{minería web, extracción de información  y descripciones semánticas}.
\item \textbf{\textit{Aktive Media}} \cite{Uren:2006} es una GUI para la descripción automática de una colección de imágenes o documentos (batch annotation) para un contexto específico. ``\textit{El objetivo de Aktive es automatizar el proceso de descripción, mediante la sugerencia del conocimiento al usuario de forma interactiva, mientras éste está describiendo}\footnote{A. Chakravarthy, V. Lanfranchi , F. Ciravegna, ``AKTive Media,''  Disponible en: \url{http://www.aktors.org/technologies/aktivemedia/index.html}}''. Estas sugerencias se hacen con base en axiomas y descripciones previas.
\end{itemize}

La finalidad de estas herramientas es facilitar la generación de descripciones en forma de triple. Sin embargo hay varias razones, por las cuáles, no se elige una de estas herramientas para alcanzar este fin. Estas razones se listan a continuación:

\begin{itemize}
\item Todas estas aplicaciones solo permiten hacer declaraciones de documentos e imágenes, por ello, no proporcionan una solución a la heterogeneidad en formato.
\item Algunas aplicaciones no interpretan los axiomas que están escritos con los vocabularios OWL y RDF(s). Herramientas con este comportamiento son: \textit{OntoMat Annotizer y MnM}.
\item Algunas aplicaciones cambian los prefijos en las tripletas por sus propios prefijos, esto significa un trabajo extra para cambiar estos prefijos con otra aplicación. Herramientas con este comportamiento son: \textit{Aktive Media y GATE}.
\item Algunas aplicaciones no tienen una versión estable. Herramientas con este comportamiento son: \textit{OntoMat Annotizer y MnM}.
\item Algunas aplicaciones no tienen documentación disponible para solucionar problemas de configuración. Herramientas con este comportamiento son: \textit{Aktive Media, GATE y MnM}.
\end{itemize}

La opción electa para representar el conocimiento explícito en forma de tripletas, es mediante el uso de scripts. Porque estos scripts facilitan y agilizan el proceso de escritura de tripletas en alguna sintaxis estándar. Un script se escribe en algún lenguaje de programación, con la finalidad de tener un mejor control en la información que se quiere representar.

Aunque estos scripts no poseen una interfaz gráfica para seleccionar la información de los recursos. Estos se puede solucionar mediante el uso de formularios web que capturen la información sobre los recursos. Posteriormente, la información se guarda en algún documento de texto plano, para que los scripts transformen  esta información en triples RDF.\\

Un \textbf{\textit{editor de ontología}} \cite{Tode} es una herramientas que proporcionan una serie de interfaces amigables para la creación y mantenimiento de ontologías. Estos editores son IDEs que permiten a los usuarios: 1)\textit{definir/visualizar las clases, propiedades, instancias y axiomas}, 2) \textit{importar y exportar las ontologías escritas con lenguajes estándar (RDF(S)y OWL)}, 3) visualizar las clases y propiedades en forma de grafo, 4) proporcionar un entorno colaborativo para el desarrollo en equipo y 5) proporcionar  soporte para el uso de inferencia en el modelo.

\textbf{\textit{Protégé}} \cite{protege} es una plataforma con un conjunto de herramientas para la creación, visualización y manipulación de ontologías en diversos formatos de representación\footnote{Stanford Center for Biomedical Informatics Research, ``Prot\'{e}g\'{e},''  Disponible en: \url{http://protege.stanford.edu/}}. Esta plataforma proporciona al usuario una interfaz amigable para la definición de clase, propiedades y axiomas, así como la introducción de datos. Protégé se puede extender sus funcionalidades a través de plug-ins y APIs. Estas funcionalidades son: 1) visualizar los axiomas, clases y propiedades en forma de grafo, 2) exportar ontologías a una variedad de formatos, 3) importar y hacer uso de algún motor de inferencia o razonador, 4) cambiar el URI del vocabulario, 5) exportar las ontología a otras sintaxis (RDF/XML, Turtle, Manchester), por mencionar algunas.

Un \textbf{\textit{triplestore}} \cite{dbpedia_2012} es un programa para \textit{el almacenamiento e indexación de tripletas RDF}, con el fin de permitir la consulta eficiente de información sobre éstas. Muchos triplestores soportan el estándar SPARQL como lenguaje de consulta para consultar el grafo RDF. Algunos de estos triplestores soportan la capacidad de inferir en el grafo RDF a partir de axiomas, mediante la incorporación o importación de un razonador para ello. Los triplestores se idealizan como \textit{sistema gestor de bases de datos para modelos basados en triplestas RDF}.

En la siguiente lista, se presentan y describen los triplestores que estudiamos.

\begin{itemize}
\item \textbf{\textit{Apache Jena}} \cite{McBride} es un \textit{marco de trabajo} la finalidad de éste es desarrollar aplicaciones que usan las tecnologías semánticas en la representación del conocimiento\footnote{The Apache Software Foundation, ``Apache Jena,''  Disponible en: \url{http://jena.apache.org/}}. Jena proporciona un conjunto de interfaces de programación de aplicaciones (API\label{sym:api}) para Java. Estas APIs tienen las siguientes funcionalidades: 1) \textit{lectura, procesamiento y escritura de triples RDF en alguna sintaxis estándar (RDF/XML, N-triples y turtle)}. 2) soporte de axiomas en los lenguajes OWL y RDF(S), 3) \textit{un motor de inferencia que soporta axiomas en OWL y RDF(S)}, 4) \textit{almacenamiento eficiente de los triples en el disco duro} y 5) \textit{un motor de consultas con soporte para el lenguaje SPARQL}.
\item \textbf{\textit{Stardog}} \cite{stardog} ``\textit{es una base de datos con todas las funciones RDF, cuyo propósito principal es ejecutar consultas en los datos RDF que están bajo su gestión directa}\footnote{Clark \& Parsia, LLC, ``Stardog,''  Disponible en: \url{http://stardog.com/}}''. Stardog soporta el lenguaje estándar SPARQL y emplea los protocolos \textit{HTTP  y SNARL} para: acceder y controlar de manera remota el modelo de datos RDF, la inferencia y el análisis de datos en el lenguaje OWL. Stardog apoyado en el motor de búsqueda de texto Lucene, proporciona la capacidad de búsquedas semánticas que consiste en la indexación de literales RDF.
\item \textbf{\textit{4store}} \cite{Fstore} \textit{es un sistema para el almacenamiento RDF que incorpora un motor de consultas SPARQL}\footnote{Garlik, ``4store,''  Disponible en: \url{http://4store.org/}}. Las principales fortalezas de esta herramienta son el rendimiento, seguridad, escalabilidad, y estabilidad.
\item \textbf{\textit{Sesame}} \cite{Sesame} \textit{es un \textit{marco de trabajo} estándar de facto para el análisis, almacenamiento, inferencia y consulta de datos RDF}\footnote{Aduna, ``Sesame,''  Disponible en: \url{http://www.openrdf.org/index.jsp}}. Este marco proporciona una API para emplea sobre distintos sistemas de almacenamiento RDF o repositorios. Finalmente, Sesame soporta: los principales sintaxis para los triples RDF y el lenguaje de consulta SPARQL.
\end{itemize}
